# Регистрация

- Ввод логина + пароля + подтверждение пароля
- Получение refresh + access токена

# Аутентификация

- Ввод логина + пароля
- Получение refresh + access токена

# Смена данных

## Вручную могут изменяться:

- Имя
- Фамилия
- Юзернейм
- Email (подумать над подтверждением)

## Без ручного участия изменяются

- tg_link и tg_id изменяются только при подключении/удалении тг аккаунта
- аналогично vk_link и vk_id (добавить потом)

## Смена пароля

    Осуществляется только через отдельную форму.
    Вводится старый пароль, новый и его подтверждение.

# Привязка TG аккаунта

## Варианты использования

1) Пользователь не авторизовывался в системе через telegram:
    - В системе нет аккаунта созданного из под telegram
    - При привязке тг аккаунта в поля tg_link и tg_id устанавливаются значения полученные от бота, аккаунт остается
2) Пользователь прошел обычную регистрацию, и также отдельно входил через tg:
    - В системе создано два аккаунта
        - После обычной регистрации
        - После авторизации через тг
    - После привязки тг (в аккаунте прошедшем обычную регистрацию) аккаунт созданный через тг удаляется, tg_link и tg_id
